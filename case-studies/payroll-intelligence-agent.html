Defining what good looks like for a high-stakes payroll agent

I led a sprint to create a 51-question diagnostic golden dataset and reusable prompt framework for a payroll agent that needed to answer complex, data-dependent customer questions safely.

Role

Golden dataset architect, model UX writer, prompt engineer, cross-functional lead.

Scope

51 high-value payroll questions, diagnostic paths, data dependencies, and source-of-truth answers.

Outcome

Delivered the full dataset in about 10 days with a reusable framework for future agent work.

Context

A partner team needed ground truth for a payroll agent, but the request was not just “write answers.” They needed a concrete definition of good behavior: what data the agent should retrieve, what it should ask the user, which root causes were plausible, and when the experience should hand off.

Payroll support is high anxiety and data-dependent. A generic answer can sound reasonable and still fail the customer if it does not diagnose the right issue.

Design shift Treat payroll support as diagnosis, not generic Q&A.

Approach

I converted the request into a diagnostic framework that product, design, content, domain experts, and agent builders could all work from.

01

Understand the question

Capture the customer question, scope, sub-variants, and critical context.

02

Map data dependencies

Separate retrievable data from must-ask questions and unsupported assumptions.

03

Define diagnostic paths

Document root causes, likely branches, tool needs, reference content, and handoff triggers.

04

Write ideal behavior

Turn domain expertise into source-of-truth answers and reusable prompt patterns.

Key decisions

Make data dependency explicit

Each answer needed to say what the system could know, what it needed to retrieve, and what it should ask the customer rather than assume.

Use a generator to scale expert judgment

I built a reusable prompt framework so domain expertise could be translated into consistent answer anatomy, not one-off prose.

Design for escalation as a feature

For regulated, account-sensitive issues, a good AI answer sometimes means knowing when not to answer and handing off with context.
