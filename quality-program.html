Case No. 03  ·  The Quality Program  ·  Agentic platform · quality team

The answer sounded right.

A fluent, confident answer invented a date about a customer's payroll taxes. I built the eval system that catches that before it ships — and turned quality into a gate every release has to pass.

Problem

Fluent answers that are confidently wrong — and read as right.

Goal

Make quality a release gate, not a subjective judgment call.

My scope

Golden datasets, LLM-judge validation, the pre-fill pipeline, and the blind audits.

Outcome

Quality became a gate every release has to clear; +8–13 pts across three skills.

01
The catch

Sounds right. Fails the bar.

Grade an answer on how it reads, and a customer gets told their taxes were filed on a day that never happened. Three gates pass; the one that matters catches the invented date.

02
Move one

I tested the judge before I trusted it.

LLM judges are cheap, fast, and unproven. I validated them against 200 human-labeled tasks first. Once they cleared the bar, the planned 2,000-row vendor labeling wasn't justified — an eval program should tell you where quality is, and where your money isn't required.

03
Move two

I made the slowest step a review, not a rewrite.

Blank-page authoring became review-and-correct across 3,000+ rows, with no added headcount. An LLM pre-fills every field; a designer reviews and corrects. The same data feeds new-skill development too.

The bug that mattered

A path could silently overwrite a human's original ground truth with model output — no audit trail, quietly poisoning every downstream judge. I caught it in review, deleted the path so originals are never touched, added a check that fails the run if it's ever violated, and shipped the fix in about four hours.

04
Move three

I made the audit blind.

Every row got a fresh KEEP-or-DELETE call with the original label hidden, so nobody could anchor on the expert's first answer. If the ground truth is wrong, everything built on it inherits the error.

05
The loop

It's a loop, not a verdict.

A model that grades how an answer reads will never catch the invented date. This loop tests for what fluent prose hides — and it runs on every skill, every release.
