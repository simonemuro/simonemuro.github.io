Case No. 03  ·  Ground truth  ·  Intuit Intelligence · quality team

Quality gated every launch. The ground truth behind it had to scale.

No skill on the platform shipped until it cleared the quality gate, and the gate ran on offline evals that scored every answer against ground truth: the ideal answer for a given scenario. As the platform grew to dozens of skills, making that ground truth by hand fell behind, one answer at a time. A gate is only as good as the answer key behind it, so the answer key had to keep up.

Problem

01
The problem

The gate was only as good as its answer key.

Quality was the gate. A skill went live, got blocked, or rolled back on its offline eval score, and every eval compared the skill's answer to ground truth: the ideal answer for a scenario, the actions it should take, and how it should be judged, on correctness, completeness, relevance, personalization, tool calls, and actions. Ground truth was the foundation the whole program stood on, and it was falling behind. Each new skill needed its own, and every answer was written from a blank sheet, one at a time. That put domain experts on data entry instead of judgment, and it left the gate unable to keep pace with the skills shipping around it.

02
The insight

Stop starting from a blank page.

The bottleneck wasn't expert judgment; it was everything wrapped around it. Experts were writing boilerplate and filling structured fields before they ever reached the part that needed them, deciding what a good answer actually was. So I moved the effort. If each answer arrived pre-drafted to spec, review and correction would replace authoring, and expert time would land where it mattered. Running the eval needed the same treatment: a dependable way to replay each conversation so the answer could be scored against ground truth.

03
The solution

A system any skill could run through.

It came together in three parts: generators that pre-fill each answer to spec, human-in-the-loop validation that puts an expert on every draft to review and correct, and simulator instructions that drive each eval by replaying the conversation to be scored. I refined the generators over many passes, each version tested on a sample before it touched the real set, until the same process could run for any skill rather than the one in front of it.

The safeguard that mattered at scale

In review, I caught a path that could quietly overwrite a validated answer with a model's draft, no trace left behind. I closed it, deleting the path and adding a check that fails the run if it is ever attempted. Every original answer stays intact, on every code path.

04
The outcome

The gate kept pace with the platform.

Offline evals ran across every skill instead of stalling behind manual authoring. A large batch that had taken about three weeks came down to about one. And on the three skills measured through this work, quality rose 8 to 13 points.
