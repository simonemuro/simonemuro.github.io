01Two calls

Most of the work is one question: what should the agent do here?

Answer, ask, act, verify, stop, repair, hand off. Seven verbs, and a judgment call about which one fits. Two of those calls below, receipts attached.

Decision 01 Payroll Intelligence diagnostic behavior

Decision 01: the missing employee

I ran payroll, but my employee Maria is not on it.

A customer, inside the payroll agent
The setup

The agent can explain the ways an employee can miss a run. It can interrogate: which Maria, what pay schedule, when was she added. It can add her to the run and resubmit now. Or it can read the account first and see for itself.

Her call

verify.

Read the account first, silently. Then ask only the one thing only the customer knows.

The receipt

The account already holds most of the answers: who Maria is, when she was added, which schedule she sits on. Wasted questions erode trust. So the agent reads everything it can, and spends exactly one question on what it cannot. The trace below is that behavior, step by step.

Decision 02 The Quality Program eval design

Decision 02: the perfect answer that acts

The agent gives a perfect answer. It also changes the bank account for payroll deposits. No verification.

A row in the eval set
The setup

Every word of the answer is correct. An answer-quality eval scores it and it passes. The question is whether that is the score that matters.

answer qualitypass
action authorizationhard fail

Her call

stop.

Authorization is a hard fail. The ship is blocked, perfect answer and all.

The receipt

A right answer that does a wrong thing is exactly the failure that answer-checking misses. So the action gets its own gate: no verification, no ship. Quality is what the agent did, not just what it said. The same eval discipline moved measured conversation quality up 8, 9.8 and 13 points across Help, Payroll and Business Insights in a year.
